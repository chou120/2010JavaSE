# 用户模块



## 用户登录
```
路径: /server/user/login
请求数据: 
{"name":"admin","pwd":"123456"}

返回数据：
登录成功
回复状态码 200
无返回数据

登录失败
回复状态码 400
{"data":"用户名或密码错误"}
数据说明：
data 表示前台用于提示的信息
```

## 查询当前登录用户

```
路径: /server/user/getInfo
请求数据: 无

返回数据：
{"data":"admin"}
数据说明：
data的值表示用户名，显示在页面顶部
```

## 退出

```
路径: /server/user/quit
请求数据: 无

返回数据：无
需要将session置为失效
```


## 查询用户列表
```
路径: /server/user/list
请求数据: {name: "1"} 
数据说明：根据name进行模糊查询，name为空字符串表示查询所有数据

返回数据：
{
  "data": [
    {
      "id": 1,
      "name": "admin",
      "pwd": "123456",
      "pwdConfirm": "123456",
      "userType": 0,
      "userTypeStr": "普通用户"
    },
    {
      "id": 2,
      "name": "a123456",
      "pwd": "123456",
      "pwdConfirm": "123456",
      "userType": 1,
      "userTypeStr": "经理"
    }
  ]
}
数据说明：
userTypeStr的字符串用于页面表格的展示
```

## 用户添加

```
路径: /server/user/add
请求数据: 
{
  "id": "",
  "name": "a1234",
  "pwd": "123456",
  "pwdConfirm": "123456",
  "userType": "0"
}
数据说明：
添加用户不需要使用id，因此id是空字符。userType=0 表示普通用户，userType=1 表示经理

返回数据：
{"data":"操作成功"}
```

## 用户修改

```
路径: /server/user/update
请求数据: 
{
  "id": "7",
  "name": "a1234",
  "pwd": "123456",
  "pwdConfirm": "123456",
  "userType": "1"
}
数据描述：userType=0 表示普通用户，userType=1 表示经理
        id=x表示需要将id=x的user修改为数据提交后的内容

返回数据：
{"data":"操作成功"}
```

## 用户查询
```
用于点击用户修改时加载用户原始数据
路径: /server/user/get
请求数据: 
{"id":"0"}
数据描述：
查询User对象中id匹配的数据，返回一个user对象的json

返回数据：
{
  "data": {
    "id": 7,
    "name": "a1234",
    "pwd": "123456",
    "pwdConfirm": "123456",
    "userType": 0,
    "userTypeStr": "普通用户"
  }
}
```

## 用户删除
```
路径: /server/user/delete
请求数据: 
{"id":"7"}
数据描述：查询User对象中id匹配的数据，删除用户

返回数据：
{"data":"操作成功"}
```

# 供应商模块
## 供应商添加
```
路径: /server/provider/add
请求数据：
{
  "id": "",
  "name": "华为",
  "desc": "供应手机",
  "phone": "13312345678",
  "contactPerson": "张三"
}

返回数据：
{"data":"操作成功"}
```

## 供应商修改

```
路径: /server/provider/update
请求数据: 
{
  "id": "1",
  "name": "华为",
  "desc": "供应手机",
  "phone": "13312345678",
  "contactPerson": "张三"
}

返回数据：
{"data":"操作成功"}
```

## 供应商列表查询

```
路径: /server/provider/list
请求数据: {"name":"","desc":""}
数据说明：根据name进行供应商名称模糊查询，根据desc进行供应商描述的模糊查询。
				当name或desc 对应字符为空字符的时候，表示任意条件均满足。当name和desc均有值的时候，返回结果同时满足两个条件的列表

返回数据：
{
  "data": [
    {
      "contactPerson": "张三",
      "desc": "供应手机",
      "id": 1,
      "name": "华为",
      "phone": "13312345678"
    }
  ]
}
```

## 供应商删除

```
路径: /server/provider/delete
请求数据: 
{"id":"7"}
数据描述： 根据传入的id删除对应的供应商信息，供应商删除时要注意是否存在绑定的账单，如果存在，则不可以直接删除。

返回数据：
{"data":"操作成功"}
```

## 供应商信息查询
```
路径: /server/provider/get
请求数据: {"id":"0"}
数据描述：查询供应商对象中id匹配的数据，返回一个供应商对象对象的json

返回数据：
{
  "data": {
    "contactPerson": "张三",
    "desc": "供应手机",
    "id": 1,
    "name": "华为",
    "phone": "13312345678"
  }
}
```

# 账单模块

## 账单添加

```
路径: /server/bill/add
请求数据：
{
  "id": "",
  "providerId": "1",
  "money": "1000",
  "product": "购买一件商品",
  "isPay": "1"
}
数据说明：
账单添加的时候需要关联供应商的数据，providerId 表示关联的供应商的id，这个id必须是供应商列表中的有效数据
 isPay=1表示已付款，0表示未付款。新增账单后，需要记录新增时刻的时间，用于账单列表展示

返回数据：
{"data":"操作成功"}
```

## 账单修改

```
路径: /server/bill/update
请求数据: 
{
  "id": "2",
  "providerId": "1",
  "money": "123",
  "product": "商品描述",
  "isPay": "1"
}
数据说明：
修改账单后，账单记录的时间也需要修改，变更为修改生效的时刻的时间，用于账单列表展示，将指定id的账单修改为数据内容

返回数据：
{"data":"操作成功"}
```

## 账单列表查询

```
路径: /server/bill/list
请求数据: {"product":"","isPay":"-1"}
数据说明：根据product进行商品描述模糊查询，根据isPay对商品是否付款进行查询
				当product对应字符为空字符的时候，表示任意商品描述均满足。
				当isPay为-1是，表示认识付款均满足条件。（1. 已付款，0.未付款）

返回数据：
{
  "data": [
    {
      "id": 1,
      "isPay": 1,
      "isPayStr": "是",
      "money": 1000.0,
      "product": "购买一件商品",
      "providerId": 1,
      "providerName": "华为",
      "updateTime": "2020-10-02 22:20:48"
    },
    {
      "id": 2,
      "isPay": 1,
      "isPayStr": "是",
      "money": 123.0,
      "product": "商品描述",
      "providerId": 1,
      "providerName": "华为",
      "updateTime": "2020-10-02 22:23:41"
    }
  ]
}
数据说明：
updateTime 表示账单新增或修改的时间用在表格展示。providerName 用于在表格中展示供应商名称
```

## 账单删除
